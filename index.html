精密仪器设计——带传动 概述

什么是带传动

利用张紧在带轮上的传动带与带轮的摩擦或啮合来传递运动和动力。

带传动的优点

结构简单，传动平稳，噪声小，可缓冲吸振。允许较大中心距。

分类————根据传动原理的不同

  1. 摩擦传动型
  2. 啮合传动型（同步带）

通常皮带传动的时候，两个皮带轮大小是不一样的，为什么两个皮带轮大小要搞的不一样呢？

速度不同，省力

我们在把机械运动的动力和速度向下级传动的时候，往往会要改变它的转速，改变运动的方向，等等，那么皮带轮的大小不同就是为了改变转速，当然在我们骑自行车的时候那么也可以为了省力。

机械运动的现象大家每天都会见到，但是可能不会每个同学都思考过它里边的一些科学问题。

带传动，它是利用张紧在带轮上的传动带与带轮的摩擦或粘合来传动运动和动力。摩擦的现象普遍存在，比如说轿车在公路上行走，那么要刹车或者要加速，都必须有车轮和路面的摩擦，如果没有摩擦或者摩擦很小，就如同滑冰一样的就会失控。

这个传动方式呢，就是运动平稳，噪音小，可以缓震缓冲或者吸收震动，这个是为什么呢？

首先，为什么会有震动？

一个概念，就是加工误差的问题。我们机械运动的动力一般来自于电动机或者柴油机内燃机。无论电动机或者柴油机，那它在生产加工的时候都会有误差。误差的存在会使得机械运动不平稳。这种不平稳导致的震动对整个机械系统是有害的。大家一定要有这么一个概念，就是没有误差，呃，加工是不存在的。没有摩擦的环境，也是很难实现的。

好了，呃，就是因为加工误差造成了极限震动，那么皮带是个柔性的，零件.它在震动的传递上非常弱，因此它可以吸收震动，减小震动。

啮合传动

那么齿轮传动是机械传动里头运用的最广泛的一种。后面的章节我们会讲到。但是齿轮传动有个非常大的局限性。就是两个齿必须挨在一起。也就是两个齿轮的轴必须要离得很近。大家可以想象一下，两个轴挨在一起和两个轴离的很远，这是两种不同的工作情形。那么我们既希望系统的传动像齿轮一样准确，又希望两根轴离得比较远。那么就要用到啮合类型的，带传动。

我们把其中一个输入运动的轮儿叫主动轮儿。跟着动的那个轮叫从动轮。以后我们看到带传动的时候，你就可以判断上一级的那个轮是主动的，下一级那轮是从动的。

大家注意这一条具有过载保护功能。为什么会有过载保护呢？

摩擦传动的这个带传动是靠摩擦力来传动的。当运动阻力过大的时候，摩擦力就不足，就会空转。这是一个非常典型的具有双刃剑的现象。具有双刃剑特点的现象。

结构简单，运动平稳，但是它就是精度低，因为他有时候会打滑。如果他没有这个打滑的现象，把运动原原本本的传下去。后面的齿轮传动，如果过载的话，那后果就相当严重了。

齿轮传动过载会发生什么现象呢？

齿轮的齿断了，那么齿轮就报废了。

所以摩擦传动这里边过载保护是非常重要的一个功能。

平皮带微型带圆带和多楔带

平皮带应用的最多了，在重型机械里边，在一般的农用机械里边很多。它是单纯的靠皮带的内侧跟带轮之间的摩擦来传动的。

如果摩擦力偏小，我想增加一下摩擦力该怎么办呢？

皮带加紧和粗糙度。都是对的，换用不同材料呢，也是对的。

  1. 皮带加紧，那么就是通过改变皮带上边的f0。大家知道摩擦力有两个因素，一个是正压力，一个是摩擦系数。改变f0，那么就要改变它上面的正压力，就把皮带张紧。
  2. 改变摩擦系数就是在皮带的内侧用皮带蜡，在皮带的内侧摩擦。皮带蜡是黑乎乎的，特别像沥青。增加了皮带和轮之间的粘性摩擦。

啮合传动型（同步带）

特点：传动比准确，速比恒定，传动比范围大，允许线速度高，传动效率可达98%，结构紧凑。

各位猜一下为什么同步内核传动大齿轮可以没有齿呢？

大家仔细看一下这个大轮和小轮。皮带在大轮上包裹的面积长度和在小轮上包裹的面积长度是有很大的不同的。如果发生打滑，只会在包裹的面积比较小的上发生。因此小轮上无论如何要有齿。

如果特别强调高精度的时候，那么就干脆两个轴都带尺吧，一般情况下大轮上没有齿也不会打滑。

map笔记

map

标准库map类型是一种以键-值(key-value)存储的数据类型

map内部的实现时它通过自建一颗红黑树，这棵树具有对数据自动排序的功能

  1. 声明方式：
    map<int,string>mapStudent;或者map<string，int>mapStudent;/两者声明方式都是正确的，但是效果是不一样的/

  2. 数据的插入

    • 第一种 

      用insert函数插入pair数据
map<int,string&gt;mapStudent;
mapStudent.insert(pair<int,string>(1,"studunt_1"));

    • 第二种 

      用insert函数插入value_type数据
map<int,string>mapStudent;
mapStudent.insert(map<int,string>::value_type(1,"student_1"))

    • 第三种

      用数组方式插入数据
map<int,string>mapStudent;
mapStudent[1] = “student_1”;
mapStudent[2] = “student_2”;

      p.s.以上三种用法，虽然都可以实现数据的插入，但是它们是有区别的，当然了第一种和第二种在效果上是完成一样的，用insert函数插入数据，在数据的插入上涉及到集合的唯一性这个概念，即当map中有这个关键字时，insert操作是不能再插入这个数据的，但是用数组方式就不同了，它可以覆盖以前该关键字对应的值,即：如果当前存在该关键字，则覆盖改关键字的值，否则，以改关键字新建一个key—value;

vector笔记

  1. 声明：

    • 一个vector类似于一个动态的动态数组

    • vector中可以存在重复的元素

    • vector<int> a;//声明一个元素为int类型的vector a

    • vector<MyType> a;//声明一个元素为MyType的vector a

      p.s:之前声明的a包含0个元素，即a.size()的值为0，但是它是动态的，其大小会随着数据的插入和删除的改变而改变

    • vector<int> a(100,a);//这里声明的是一个已经存放了100个0的整数的vector

  2. 向量操作

  • 常用函数

    • a.size();//返回vector的大小，即包含的元素个数
    • a.pop_back();//删除vector末尾的元素，vector的大小相应减1
    • a.push_back();//用于在vector的末尾添加元素
    • a.back();//返回vector末尾的元素
    • a.clear;//将vector清空，vector的大小变为0

  • 其他访问方式

    • cout<<a[5]<<endl;
    • cout<<a.at(5)<<endl;

      以上的区别在于后者在访问越界时会抛出一场，而前者不会

  1. 函数汇总
    • size() 得到vector的大小，即包含的元素个数
    • pop_back() 删除vector末尾的元素，vector的大小相应减1
    • push_back() 用于在vector的末尾添加元素
    • back() 得到vector末尾的元素
    • clear() 将vector清空，vector的大小变为0
    • at() 得到编号位置的数组
    • begin() 得到数组头的指针
    • end() 得到数组的最后一个单元+1的指针
    • front() 得到数组头的引用
    • max_size() 得到vector最大可以是多大
    • capacity() 当前vector分配的大小
    • resize() 改变当前使用数据的大小，如果他比当前使用的大，则填充默认值
    • reserve() 改变当前vector所分配的空间大小
    • erase() 删除指针指向的数据项
    • rbegin() 将vector反转后的开始指针返回(其实就是原来的end-1)
    • rend() 将vector反转后的结束指针返回(其实就是原来的begin-1)
    • empty() 判断vector是否为空
    • swap() 与另一个vector交换数据

html

什么是html？

HTML 是用来描述网页的一种语言。
HTML 指的是超文本标记语言 (Hyper Text Markup Language)
HTML 不是一种编程语言，而是一种标记语言 (markup language)
标记语言是一套标记标签 (markup tag)
HTML 使用标记标签来描述网页

那么什么是html的标签

HTML 标记标签通常被称为 HTML 标签 (HTML tag)。
HTML 标签是由尖括号包围的关键词，比如 <html>
HTML 标签通常是成对出现的，比如 <b> 和 </b>
标签对中的第一个标签是开始标签，第二个标签是结束标签
开始和结束标签也被称为开放标签和闭合标签

反正我理解的html标签大概用<>标注起来的东东

空的html元素

空的 HTML 元素
没有内容的 HTML 元素被称为空元素。空元素是在开始标签中关闭的。
<br> 就是没有关闭标签的空元素（<br> 标签定义换行）。
在 XHTML、XML 以及未来版本的 HTML 中，所有元素都必须被关闭。
在开始标签中添加斜杠，比如 <br />，是关闭空元素的正确方法，HTML、XHTML 和 XML 都接受这种方式。
即使 <br> 在所有浏览器中都是有效的，但使用 <br /> 其实是更长远的保障。

属性总是在 HTML 元素的开始标签中规定
